# geo-sentiment-optimizer

## 目标

审计品牌内容中的情感信号，识别会被 AI 平台引用放大的负向风险与缺失的正向信号，并给出按优先级排序的改写建议，从而改善 AI 搜索与问答平台对品牌的整体刻画。AI 对品牌的情感倾向由其抓取到的内容决定——优化发布内容即可改善 AI 的品牌印象。

## 使用场景

- 用户反馈「AI 在说我们品牌的坏话」「AI 对我们品牌的描述不准确」时，对品牌页面内容做情感审计。
- 文章生成或改写完成后，作为「情感优化」子步骤独立调用，检查正向 / 负向 / 中性信号覆盖情况。
- 需要重写首页、定价页、关于页等关键页面文案，以强化正向品牌联想时调用。

## 输入

- `content`: 需要审计的品牌内容文本（如首页、关于页、产品页的文案）。
- 可选：品牌域名 / 品牌名，用于报告标题与上下文标注。

在 Electron 主进程中通过 TypeScript 函数调用（替代原 Python CLI）：

```ts
import {auditSentiment} from './scripts/audit_sentiment.ts';

const result = auditSentiment(content);
// => { positive: 2, negative: 1, score: 6 }
```

## 输出

`auditSentiment()` 返回 `SentimentAuditResult`：

```json
{
  "positive": 2,
  "negative": 1,
  "score": 6
}
```

- `positive`：命中的正向信号数量（0-3）。
- `negative`：命中的负向信号数量（0-3）。
- `score`：情感得分，`5 + positive - negative`。

基于审计结果，Agent 应产出完整的情感审计报告，结构如下：

```markdown
# 品牌情感审计 — [品牌]

**情感风险等级**：[高 / 中 / 低]

## 正向信号：[n]/10 ✅
✅ 清晰的价值主张
❌ 缺少奖项 / 媒体认可 → 增加媒体报道页

## 负向风险：[n] 🔴
🔴 未回应的负面评价 → 撰写回应内容
🔴 含糊的局限性表述 → 直接说明限制

## 优先级改写建议
1. **首页** → 用具体成果替代笼统宣称
2. **定价页** → 增加社会证明与信任信号
3. **关于页** → 补充创立故事与资质

## 优化后文案
[改写后的段落，强化正向情感信号]
```

## 情感信号方法论

AI 平台从以下来源综合形成品牌情感：

1. **品牌自有内容** — 品牌如何描述自己
2. **第三方报道** — 评测、媒体、社交提及
3. **结构化数据** — 评分、评价 schema
4. **对批评的回应** — 品牌如何处理问题
5. **对比内容** — 与竞品对比时的表述

### 正向信号 ✅

| 信号 | 检查点 | 缺失时的修复 |
|------|--------|--------------|
| 价值主张 | 首段清晰表达 | 重写开头 |
| 客户成果 | 有具体数字 / 结果 | 补充案例研究 |
| 社会证明 | 客户数、评分、媒体报道 | 增加证明材料 |
| 奖项 / 认可 | 明确具名 | 建立认可专区 |
| 信任信号 | 创立时间、团队规模、融资 | 补充到关于页 |

### 负向信号 🔴

| 信号 | 风险 | 修复 |
|------|------|------|
| 未回应的负面评价 | AI 引用投诉内容 | 撰写回应内容 |
| 含糊的局限性表述 | 显得回避问题 | 直接说明限制 |
| 缺少投诉类 FAQ | 常见问题无人解答 | 增加 FAQ 版块 |
| 信息过时 | 使 AI 产生混淆 | 保持内容更新 |
| 对比页弱势 | 突出竞品优势 | 强化自身定位 |

### 中性信号 🟡

| 信号 | 问题 | 解决方案 |
|------|------|----------|
| 「行业领先」类宣称 | 笼统、无法验证 | 替换为具体事实 |
| 被动语态 | 软弱、不清晰 | 改用主动语态 |
| 无客户证言 | 缺乏社会证明 | 增加客户故事 |

### 改写规则

1. **具体证据优于笼统宣称**：❌「我们是最好的 AI SEO 工具」→ ✅「GEOly AI 在 5 个 AI 平台上监测 1,000+ 品牌」。
2. **主动回应负面问题**：在 FAQ 中直接回答常见质疑，用具体客户案例佐证（如「Starter 版 $29/月，专为 10 人以下团队设计」）。
3. **以成果为导向的描述**：❌「平台有 50+ 功能」→ ✅「使用 GEOly AI 的品牌在 90 天内 AIGVR 得分提升 23 分」。
4. **建立正向实体联想**：奖项（「2024 年 XX 奖得主」）、客户（「受 XX 等知名公司信赖」）、媒体（「曾被 XX 报道」）。

## 约束

- 纯只读 / 纯生成操作，不修改任何已有数据，审计与改写建议仅输出为报告。
- 关键词信号检测是确定性启发式方法，仅作初筛；最终审计报告需结合信号方法论由 Agent 综合判断。
- 改写建议必须用具体事实（数字、案例、具名来源）替代笼统宣称，不得编造无法验证的成果数据。
- 不处理第三方平台内容（评测站、社交媒体），仅优化品牌自有可控内容。

## Domain 差异

本 Skill 适用于所有 domain（`domains` 为空列表），无特化差异。情感信号框架（正向 / 负向 / 中性）与改写规则对本地服务、SaaS、电商品牌通用，差异仅体现在具体案例与信任信号的类型上（如本地服务侧重门店评价与口碑，SaaS 侧重客户成果数据，电商侧重评分与退货政策）。
